The problems

  • Multiple SOLID violations
    • S: authorization check, validation, business logic and data persistence in a single method
    • O: hardcoded checks (e.g., user role comparison and error responses)
    • D: directly depends on auth() and request()
  • Testing
    • can only be tested with an HTTP call
  • Complexity
    • too many returns and conditions
    • hard to understand

The problems are endless...

  • Magic strings
    • error messages as plain text
  • Code duplication
    • repeated use of request input read
    • hard to maintain
  • Error handling
    • no valid HTTP status code used
    • need unnecessary checks on client side
  • API versioning
    • can't make change unless breaking the current code
